# 마우스 호버시 텍스트 기울기

## 알고 있는 것 & 새롭게 알게 된 것

- 유튜버님이 짜는걸 보고 있으면 불필요한 속성이 몇개 보인다. 아마 이 당시 영상 찍을 때는 CSS에 대한 숙련도가 높으시진 않으셨던거 같다. 예를 들면 `inline-block`으로 가로로 나열되게 하였는데 불필요하게 `float`를 사용하는 경우에서 그런 것들을 느끼게 되었다. 그런데 효과에 관해서는 확실히 많이 알고 계신거 같아 많이 배우고 있다.

- `inline-block`으로 자식요소를 나열하는 경우 요소들 사이 사이에 미세한 여백이 존재하게 된다. 이 여백을 지우는 방법은 다양한데 나는 부모 요소에 `font-size: 0`을 주는 방식을 사용했다. 이런 경우 자식 요소에 `font-size`를 주어야 된다.

- 부모요소에 `text-align: center`가 있는 경우 자식 요소에 `inline, inline-block`이 있으면 글자처럼 취급된다. 그러므로 `text-align`의 영향을 받는다.

- `height` 요소와 `line-height` 요소가 동일하면 세로 중앙 정렬이 된다.

- 가상 클래스(:hover, :active)에서 효과가 일어나는 경우 해당하는 요소에 `transition`을 지정해야한다.

- `nth-child`는 1번부터 시작한다.

- `shewY`는 기울기와 관련된 요소이다. 각각 X, Y축으로 기울기를 설정한다.

- `border-radius`의 경우 `border-top-left-radius`이런 식으로 4 모서리의 `radius` 값을 각 각 설정할 수 있다.

- `rotateX`는 정면을 기준으로 회전을 한다고 표현해야할까? 이게 참 효과란 것을 말로 표현하기가 참 힘들다.

- 박스 안쪽에 그림자를 표현해야 한다면 `inset` 이라는 속성값을 사용한다.
