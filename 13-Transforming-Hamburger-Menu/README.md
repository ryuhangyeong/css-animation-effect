# 변환되는 햄버거 메뉴

## 영상을 보기에 앞서 생각해보기

- 세개의 막대가 있다. 이 막대를 만드는건 태그 3개를 쓰는 방법도 있을거고 가상 요소(`::after`, `::before`)를 쓰는 방법도 있을 것이다. 음.. 내가 만든다면 가상 요소를 알고 있으니까 가상요소를 쓸거 같긴 하다. 물론 위치 조정을 일일이 해줘야해서 조금 귀찮긴 하다.

- 그리고 앞서 배운 `transition-origin`을 이용하여 기준점을 잡고 `rotate`를 이용하여 회전을 하면 될 것이다. 그리고 중간에 있는 막대는 `opaticy: 0`을 이용하면 될거 같다.

## 알고 있는 것 & 새롭게 알게 된 것

- `rgba`와 `opacity`
  opacity 는 부모요소에 적용시키면 자식 노드에게도 모두 상속되고 자식 노드에서만 opacity 를 해제(제거)할 수 없기 때문에 배경과 텍스트를 분리할 수가 없는 문제에 봉착하게 됩니다. alpha 값은 자식노드에게 영향을 미치지 않기 때문에 텍스트와 이미지,백그라운드컬로등을 분리하여 사용할 수 있습니다.

- `transparent`
  투명한

- 일단 내가 생각했던 방법은 반은 맞고 반은 틀렸다. 대략적인 아이디어는 맞았지만 실제 생각한대로 구현해보니 모양이 이상했다. 내가 생각한 속성으로 구현을 하는것은 맞았지만 방향이 조금 달랐다. 분석해보자.

  일단 기본적으로 `transform-origin`의 기본값은 현재 위치다. 현재 위치에서 `rotate`를 하면 현재 놓아져있는 위치에서 회전한다. 1번째와 3번째 요소에 `rotate`를 준다. 근데 여기에서 중요한 것은 단순 `rotate`만 주면 안된다는 것이다. X축 100% / Y축 0으로 하면 모양이 틀어진다. 난 이렇게 하면 딱 원하는 모양이 나올줄 알았는데 아니였다..

  원점으로 돌아와서 일단 `transform-origin`는 앞서 설명한대로 X축 100% / Y축 0가 아닌 center로 하고 `translateY`값을 올리고 내리면 된다. 근데 여기에서 얼마나 내리고 올려야할까를 고민해아야 한다.

  영상에서는 계산 방법에 대해서 나오지 않아 내가 유추를 해야할 듯하다. 내 추측이지만 한 번 적어본다. 일단 막대 하나당 `8px`이고 각 각의 막대는 `margin: 16px` 거리를 가지고 있다. 1, 3번째 막대의 회전 축이 정가운데 위치하려면 결국 `8 + 16`만큼 내려오거나 올라와야한다는 것을 의미한다. 이 가설을 증명하기 위해 이 공식을 대입하여 값을 변경했더니 정확히 일치한다.
